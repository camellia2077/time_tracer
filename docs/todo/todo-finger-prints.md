主要说一下流程，就是手机端首次导出的时候，必须设置一个“主口令”，即便他们打算以后只用指纹。这是因为指纹只能在“这台设备”上有效，它带不走。如果没有口令，这份数据传到 Windows 上就是一堆废铁。


设置口令（强制环节）：
App 提示：“请设置一个加密口令，用于在 Windows 端查看或在更换手机时恢复数据。”
用户输入：MySecretPassword。
生成内核资产：
程序生成随机的 Master Key。
程序用 MySecretPassword 处理后的密钥，把 Master Key 加密后存入文件头（这保证了 Windows 能开）。
开启指纹（快捷环节）：
App 接着问：“是否开启指纹，以便在本机快速解锁？”
用户选“是”并刷指纹。
硬件加固：
程序在 TEE 里生成硬件密钥，把刚刚生成的 Master Key 额外加密一份存起来
