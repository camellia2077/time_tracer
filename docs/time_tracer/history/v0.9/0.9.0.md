## [v0.9.0] - 2026-03-23

### 新增功能 (Added)
* 新增 `docs/time_tracer/core/ingest/day_bucket_and_wake_anchor_semantics.md`，明确 `day bucket`、`wake anchor`、自动补出的 `sleep_night` 活动与 `>24h` 单日时长的当前业务口径。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs` 目录命名：将 `libs/tracer_core/src/infrastructure`、`libs/tracer_core/tests/infrastructure`、`libs/tracer_core/infrastructure` 与 `libs/tracer_adapters_io/src/infrastructure` 统一重命名为 `libs/tracer_core/src/infra`、`libs/tracer_core/tests/infra`、`libs/tracer_core/infra` 与 `libs/tracer_adapters_io/src/infra`，并同步更新构建脚本、测试引用与文档路径口径。
* 重构 `libs/tracer_core/src/application/parser/**`、`libs/tracer_core/src/domain/logic/converter/convert/core/**` 与 `libs/tracer_core/src/application/parser/memory_parser.cpp`，将 ingest 解析阶段收敛为“原始事件 + 日级元数据”主路径，不再在解析过程中固化学习/睡眠/娱乐/运动等整包派生统计。
* 重构 `libs/tracer_core/src/application/parser/text_parser.cpp`、`libs/tracer_core/src/domain/model/time_data_models.hpp` 与 `libs/tracer_core/src/application/importer/model/import_models.hpp`，把 `wake anchor` 明确收敛为由 `getupTime + isContinuation` 推导的日级状态，并将 `days.wake_anchor` 作为稳定持久化元数据写入。
* 重构 `libs/tracer_core/src/domain/logic/converter/convert/core/converter_core.cpp` 与跨天补链语义，明确自动补出的 `sleep_night` 只属于 activity fact，不再承担 day status 来源；生成睡眠路径仅控制事实补链，不改变 `wake_anchor` 判定。
* 重构 `libs/tracer_core/src/infra/reports/data/utils/time_derived_stats.hpp` 与 `libs/tracer_core/src/infra/reports/data/queriers/**`，将学习、睡眠、运动、娱乐等派生统计下沉到 query/report 阶段按事实数据现算，统一从 `processedActivities` 与持久化日元数据重建报表统计口径。

### 修复 (Fixed)
* 修复解析阶段同时承担“事实生成”和“派生统计固化”导致的语义耦合问题，避免 `wake keyword`、`isContinuation`、自动补睡眠活动与日状态字段之间互相混淆。
* 修复 `wake_anchor` 与任意 `sleep_*` 活动混为同一语义来源的问题，保证报表头 `Wake Anchor`、`days.wake_anchor` 与 `Wake Anchor Days (True)` 只依赖日元数据，不依赖是否存在自动补出的睡眠事实。
