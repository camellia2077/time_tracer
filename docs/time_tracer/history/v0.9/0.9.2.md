## [v0.9.2] - 2026-03-24

### 新增功能 (Added)
* 新增 `apps/tracer_core_shell/api/c_api/capabilities/{pipeline,query,reporting,exchange,config}/**`、`apps/tracer_core_shell/api/c_api/runtime/**`、`apps/tracer_core_shell/host/bootstrap/**` 与 `apps/tracer_core_shell/host/exchange/**`，把 shell 层显式分成 capability facade、runtime glue、bootstrap 与 exchange bridge 家族。
* 新增 `shell_aggregate` verify/build profile，以及 `test/suites/tracer_windows_rust_cli/config_shell_aggregate.toml` 与 `tests/commands_shell_aggregate.toml`，让 non-owner shell facade 拥有独立的 focused verify 入口。
* 新增 `tools/toolchain/commands/cmd_quality/verify_internal/verify_profile_inference.py` 与 `tools/toolchain/ci/capability_smoke_profiles.py`，统一本地 `verify` 与 CI changed-path gate 的 capability profile 推断规则。
* 新增 `apps/tracer_core_shell/tests/integration/tracer_core_c_api_{query,reporting,pipeline}_tests.cpp` 与对应 `*_test_main.cpp`，以及 `android_runtime_reporting_error_report_tests.cpp`、`file_crypto_exchange_test_main.cpp`，把 shell 侧 capability-focused 回归入口落成 checked-in targets。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/tracer_core_shell/api/c_api/CMakeLists.txt` 与 `apps/tracer_core_shell/api/android_jni/CMakeLists.txt`，把 C API / JNI 的源码归属对齐到 `capabilities/**`、`runtime/**`、`host/bootstrap/**` 与 `host/exchange/**` 新路径族。
* 重构 `apps/tracer_core_shell/cmake/CoreTargets.cmake`，把原来的聚合 C API stability 面与 file-crypto 大桶测试拆成 `tc_c_api_{query,reporting,pipeline,shell_aggregate}_tests`、`tt_exchange_runtime_tests` 与 `tt_file_crypto_runtime_bridge_tests`。
* 重构 `tools/toolchain/config/build.toml`、`tools/toolchain/commands/cmd_quality/verify.py`、`verify_native_runner.py`、`verify_profile_policy.py` 与 `tools/tests/groups.py`，让 `verify --app tracer_core_shell` 在未显式传 `--profile` 时支持 capability-first 路由，并把 focused profile 的 native target、CLI suite 与 Python 自测入口统一到同一套配置。
* 重构 `.github/workflows/windows-build-matrix.yml`，在现有 heavy verify 之前新增 `windows-capability-smoke` 前置 gate，按 changed paths 选择 `cap_*` 或 `shell_aggregate` focused smoke。
* 重构 `apps/tracer_core_shell/tests/platform/infrastructure/tests/android_runtime/android_runtime_validate_logic_structure_reporting_tests.cpp`，拆成 `android_runtime_pipeline_validation_regression_tests.cpp` 与 `android_runtime_reporting_error_report_tests.cpp`，让 pipeline regression 与 reporting error-report 回归按真实 owner 路由。

### 修复 (Fixed)
* 修复 `apps/tracer_core_shell/tests/integration/tracer_core_c_api_{query,reporting}_test_main.cpp` 在拆出 focused target 后缺少 baseline ingest 的问题，使 query/reporting C API 回归不再依赖已删除的聚合 stability 入口隐式准备数据库。
* 修复 `apps/tracer_core_shell/tests/integration/tracer_core_c_api_{query,reporting,pipeline}_test_main.cpp` 的 runtime/dll 生命周期顺序，避免 focused target 在 `RuntimeGuard` 尚未析构时提前卸载 `tracer_core.dll`。
* 修复 `test/suites/tracer_windows_rust_cli/tests/commands_shell_aggregate.toml` 的场景归属，移除误带的 query tree 成功路径，只保留 shell aggregate 应承担的版本、帮助与 CLI failure contract。
* 修复 `tools/toolchain/commands/cmd_quality/verify_internal/verify_profile_inference.py` 的 worktree rename 解析与 path normalization，使默认 `verify` 在存在重命名改动时仍能正确识别 focused profile 或给出明确的 `fast` fallback 原因。
* 修复 `test/golden/report_formatter_parity/v1/day.md` 与当前 Markdown formatter 输出的换行差异，使 `tt_fmt_parity_tests` 与 `cap_reporting` focused verify 恢复稳定通过。

### 弃用/删除 (Deprecated/Removed)
* 删除 `apps/tracer_core_shell/api/c_api/tracer_core_c_api_workflow.cpp`、`tracer_core_c_api_reporting.cpp`、`tracer_core_c_api_crypto.cpp`、`tracer_core_c_api_internal.cpp` 与 `c_api_parse_bridge.cpp` 旧平铺路径，改由 `api/c_api/capabilities/**` 与 `api/c_api/runtime/**` 承载。
* 删除 `apps/tracer_core_shell/host/android_runtime_factory*`、`android_runtime_config_bridge.*`、`native_bridge_crypto_calls.cpp`、`crypto_progress_bridge.*` 与 `tracer_exchange_inspect_formatter.*` 的旧根路径布局，改由 `host/bootstrap/**` 与 `host/exchange/**` 承载。
* 删除 `tc_c_api_stability_tests` 与 `tt_file_crypto_tests` 两个 shell 侧混责目标，改由 capability-focused / shell-aggregate-focused targets 取代。
