## [v0.9.3] - 2026-03-26

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/tracer_core/src/domain/reports/**`、`libs/tracer_core/src/infra/reporting/**`、`libs/tracer_core/src/infra/query/**` 与 `libs/tracer_transport/src/runtime_codec_tree.cpp` 的时长、项目 ID 与聚合统计口径，把分散的 `long long` 统一收敛为 `std::int64_t`，统一 core reporting/query/transport 的数据契约。
* 重构 `libs/tracer_core/src/infra/reporting/data/utils/project_tree_builder.{hpp,cpp}`、`libs/tracer_core/src/infra/reporting/data/queriers/{daily,monthly,weekly,yearly,period}/**` 与 `libs/tracer_core/src/infra/reporting/shared/formatters/**`，统一 `BuildProjectTreeFromIds`、`project_stats`、`project_agg`、`duration_seconds` 与格式化链路的签名和容器类型。
* 重构 `libs/tracer_core/src/application/use_cases/{pipeline_api,tracer_core_api_helpers,tracer_exchange_api}.cpp`，移除宽泛的 `using namespace tracer_core::core::dto`，改为显式 DTO 导入，收敛 use case 层的命名可见域。
* 重构 `libs/tracer_adapters_io/src/infra/io/core/{file_reader,file_writer}.cpp` 与多处 reporting/shared helper 的局部变量命名，统一常量式临时值和格式化辅助实现的风格口径。

### 修复 (Fixed)
* 修复上一轮 core/reporting 类型调整后未同步覆盖 Android/JNI 消费链路的问题，使 `libs/tracer_core` 到 `libs/tracer_transport` 的报告树、时长字段与项目聚合结果改为稳定的 `std::int64_t` contract，补齐 Android 侧 native bridge 可安全复用的跨平台整数口径。
* 修复 `libs/tracer_core/src/infra/platform/windows/windows_platform_clock.cpp`、`libs/tracer_core/src/shared/utils/period_utils.cpp` 与 `libs/tracer_core/src/infra/reporting/shared/utils/format/time_format.{hpp,cpp}` 的平台相关整数换算口径，避免继续混用 `long long` 导致不同平台编译/桥接语义不一致。
