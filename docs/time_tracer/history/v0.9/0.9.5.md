## [v0.9.5] - 2026-04-02

### 新增功能 (Added)
* 新增 reporting 错误契约字段 `error_contract`，为 `day/week/month/year` 目标不存在场景统一输出稳定错误码 `reporting.target.not_found`、错误分类与提示信息。
* 新增 `report_window_metadata` 窗口元数据，并在 `recent` 与 `range` 查询结果中补充 `has_records`、`matched_day_count`、`matched_record_count`、`start_date`、`end_date`、`requested_days`。
* 新增 `libs/tracer_transport` 响应信封对 report window metadata 的序列化与反序列化支持，确保运行时桥接可以透传空窗口查询结果。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/tracer_core` reporting DTO 与公共 envelope，统一让文本输出、结构化输出与目标列表输出携带 `error_contract`，并让 `recent`、`range` 文本格式化链路附带 `report_window_metadata`。
* 重构 `libs/tracer_core/src/application/use_cases/report_api.cpp` 的查询异常处理，改为显式捕获 `ReportingContractError`，保证 reporting 语义错误不会退化为普通运行时失败。
* 重构 `recent` 参数解析逻辑，改为通过 `ParseRecentDaysArgument` 统一校验正整数输入，避免不同查询入口出现不一致行为。
* 重构 `RangeQuerierBase` 的统计补全逻辑，在区间查询完成后统一回填匹配天数、匹配记录数与是否存在记录，作为 `recent`、`range` 空窗口输出的标准口径。

### 修复 (Fixed)
* 修复 `day` 查询在目标日期不存在时仍返回普通失败文本的问题，改为抛出并透出 `reporting.target.not_found` 契约错误。
* 修复 `week`、`month`、`year` 查询在无对应数据时与 `recent`、`range` 语义混淆的问题，改为对命名目标缺失直接报错，而不是返回空报表。
* 修复 `recent`、`range` 查询在无数据窗口时缺少显式统计信息的问题，改为返回成功结果，并明确标记 `has_records=false`、`matched_day_count=0`、`matched_record_count=0`。
* 修复 transport 层无法表达 report window metadata 的问题，避免上层丢失空窗口查询的边界信息。
