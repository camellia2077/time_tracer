## [v0.9.6] - 2026-04-07

### 新增功能 (Added)
* 新增 `docs/time_tracer/core/capabilities/config/alias_mapping_rules.md`，集中说明 alias key、canonical activity path、child file 边界与顺序语义，并明确 alias mapping 只负责活动名归一化。
* 新增 canonical/runtime `converter/aliases/*.toml` 目录布局，把 alias 规则按顶层 `parent` 拆分到不同 TOML，减少右值中重复书写的 `parent` 信息，并让后续新增活动时可以按不同 TOML 维护。

### 技术改进/重构 (Changed/Refactor)
* 重构 `assets/tracer_core/config/converter/alias_mapping.toml` 为 index 文件，改由 `includes = [...]` 引用 `aliases/*.toml`；同一 child file / group 内的 alias 项书写顺序不再承载语义，只影响可读性。
* 重构 `libs/tracer_core` converter config loader / validator，改为读取 alias index 与 child files 后展开为平铺 `text_mapping`，并统一执行：左键值必须全局唯一可确定、右键值不要求唯一、重复左键严格禁止。
* 重构 active converter config 安装链路为整体 bundle 替换，统一复制 `main + alias index + duration + aliases/`，不再为小体量文本配置引入增量 diff / merge 逻辑。
* 重构 tracer exchange 包布局到 `package_version = 4`，引入 `alias_mapping_index + alias_mapping_files` manifest 字段，使导入/导出/inspect 可以完整携带按 `parent` 拆分后的 alias bundle。
* 重构 canonical bundle、平台 config sync 与 Windows CLI exchange fixture，使 runtime config、exchange package 与 config refresh 回归统一使用新的 `aliases/*.toml` 结构。
* 重构 TXT month-file day-block 能力边界，把 `default_day_marker`、`resolve_day_block`、`replace_day_block` 下沉到 `tracer_core` pipeline/runtime，并通过统一 TXT runtime contract 暴露给 Android 与 Windows CLI 复用；Android 保留 UI 状态与交互编排，CLI 保留命令解析、文件读取与 host 侧错误翻译，不再各自持有一套 day-block 业务语义。

### 修复 (Fixed)
* 修复 alias 配置持续增长后 `parent` 信息在右值中重复出现、难以按类别定位和新增活动的问题，改为按 `parent` 拆分存储。
* 修复 exchange 导出、导入与 inspect 仍按旧固定 3 个 converter TOML 处理配置的问题，避免新 alias 目录结构下丢失 `aliases/*.toml` 子文件。
* 修复 runtime config 安装、回滚与测试夹具仍假设只有单个 `alias_mapping.toml` 的问题，保证按 `parent` 拆分后的 alias bundle 能被完整复制、备份、恢复与断言。
