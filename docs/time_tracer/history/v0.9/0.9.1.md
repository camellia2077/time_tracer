## [v0.9.1] - 2026-03-24

### 新增功能 (Added)
* 新增 `libs/tracer_core/src/application/dto/pipeline_requests.hpp`、`query_requests.hpp`、`query_responses.hpp`、`reporting_requests.hpp`、`reporting_responses.hpp`、`exchange_requests.hpp`、`exchange_responses.hpp` 与 `shared_envelopes.hpp`，把 `tracer_core` 的请求/响应模型收敛为 capability-owned DTO family。
* 新增 `libs/tracer_core/src/application/aggregate_runtime/**`、`libs/tracer_core/src/application/compat/reporting/**`、`libs/tracer_core/src/application/runtime_bridge/**`，明确 aggregate runtime、reporting retained compat 与 shell/runtime bridge 的物理归属。
* 新增 `libs/tracer_core/tests/application/tests/application_aggregate_runtime_smoke_tests.cpp`、`application_pipeline_module_smoke_tests.cpp`、`application_query_tree_module_smoke_tests.cpp`、`application_workflow_module_smoke_tests.cpp`，以及 `libs/tracer_core/tests/infra/tests/infrastructure_modules_smoke_*_main.cpp`，把 active smoke tests 切成 capability / non-owner 入口。
* 新增 `libs/tracer_core/src/infra/query/data/repository/query_runtime_service*.{cpp,hpp}`、`src/infra/query/data/internal/project_tree_projection.hpp` 与 `src/infra/query/data/renderers/detail/query_duration_text.hpp`，把 query read-side 运行时实现与树投影/文本渲染辅助拉回 query owner 路径。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/tracer_core/src/application/pipeline/pipeline_workflow.cpp`，把 pipeline 支撑逻辑下沉到 `libs/tracer_core/src/application/pipeline/detail/pipeline_workflow_support_impl.inc` 与 `pipeline_replace_month_support_impl.inc`，让主文件只保留编排职责。
* 重构 `libs/tracer_core/src/infra/exchange/tracer_exchange_service_common.cpp` 与 `tracer_exchange_service_import.cpp`，把 progress/security 映射、package 文件系统辅助、converter config 应用、import transaction 支撑拆到 `libs/tracer_core/src/infra/exchange/detail/**`。
* 重构 `libs/tracer_core/src/application/use_cases`、`application/reporting`、`infra/query`、`infra/reporting` 与 `infra/exchange` 的 include 入口，统一消费 capability-owned DTO 与新路径族，消除对旧 umbrella DTO 和模糊 owner 路径的依赖。
* 重构 `libs/tracer_core/tests/application/tests/modules/*.hpp` 与 `libs/tracer_core/tests/infra/tests/modules_smoke/*.hpp`，让 application / infra 测试声明按 capability 路由，不再通过跨 capability 总索引头聚合。
* 重构 `libs/tracer_core/src/infra/reports/**`、`src/infra/modules/reports/**` 与相关 CMake/source list，统一改名为 `infra/reporting/**` / `infra/modules/reporting/**`，让 reporting 的实现树、module surface 和 capability 命名保持一致。
* 重构 `libs/tracer_core/src/infra/persistence/sqlite_data_query_service*.{cpp,hpp}` 到 `src/infra/query/data/repository/query_runtime_service*`，把 query read-side runtime service 从 persistence 边界移回 query capability。
* 重构 `libs/tracer_adapters_io/src/infra/io/internal/runtime_adapter_types.hpp` 与 `src/modules/tracer.adapters.io.runtime.cppm`，让 adapters runtime concrete declarations 跟随新的 runtime bridge/module-first 布局。

### 修复 (Fixed)
* 修复 `libs/tracer_core/src/application/pipeline/pipeline_workflow.cpp` 的 direct conversion failure 语义，失败时会继续保留 buffered diagnostics、写入已启用的 error report writer，并把 `Full error report` 路径带回顶层错误消息。
* 修复 `libs/tracer_core/src/application/parser/text_parser.cpp` 与 pipeline logic validation 的 source span 传递链路，使结构/逻辑诊断继续保留可点击的 `path:line` 信息。
* 修复 `libs/tracer_core/tests/infra/tests/modules_smoke/logging_platform_config.cpp`、`query_internal_orchestrators.cpp`、`query_stats_repository.cpp`、`reports.cpp` 等 smoke tests 对新 capability 边界的适配，避免边界重构后出现假回归。

### 弃用/删除 (Deprecated/Removed)
* 删除 `libs/tracer_core/src/application/dto/core_requests.hpp`、`core_responses.hpp`、`tree_query_response.hpp` 旧 umbrella/alias DTO 入口，active code 改为直接依赖 capability-owned DTO 或 `application/dto/compat/**`。
* 删除 `libs/tracer_core/src/application/interfaces/i_report_handler.hpp`、`i_report_exporter.hpp`、`i_report_query_service.hpp` 与 `libs/tracer_core/src/application/use_cases/i_tracer_core_runtime.hpp`、`tracer_core_runtime.hpp`、`tracer_core_runtime.cpp` 旧模糊路径，改由 `application/compat/reporting/**` 与 `application/aggregate_runtime/**` 承载。
* 删除 `libs/tracer_core/tests/application/tests/application_modules_smoke_tests.cpp`、`tests/application/tests/test_main.cpp`、`tests/application/tests/modules/test_modules.hpp`、`tests/infra/tests/infrastructure_modules_smoke_tests.cpp` 与 `tests/infra/tests/modules_smoke/persistence.cpp` 老大桶测试入口。
* 删除 `libs/tracer_core/src/application/use_cases/tracer_core_api*.{cpp,hpp,module.cpp}` 与 `libs/tracer_core/src/infra/modules/reports/**` 旧聚合 module / 旧 capability 命名族，active code 改为直接进入 capability-owned API、DTO 与 `infra/modules/reporting/**`。
