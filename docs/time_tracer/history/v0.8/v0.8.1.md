## [v0.8.1] - 2026-03-09

### 新增功能 (Added)
* 新增 `python tools/run.py validate` 作为通用编译/测试入口，统一支持 `--plan`、`--paths` / `--paths-file`、结构化 `summary.json` 与精简/全量日志分离输出。
* 新增 `libs/tracer_core/src/application/pipeline/` 的 `types`、`stages`、`orchestrator` 模块族，建立 `tracer_core` 第一批 “拥有声明 + 拥有实现” 的 C++20 module family 模板。
* 新增 `apps/tracer_cli/windows/rust_cli/src/cli/*.rs` 与 `apps/tracer_cli/windows/rust_cli/src/core/runtime/callbacks/` 职责拆分结构，补齐 Rust CLI 命令解析与运行时回调的可维护边界。

### 技术改进/重构 (Changed/Refactor)
* 重构 `tools/toolchain/**` 与 `.agent/rules/AGENTS.md`，将旧 `post-change` 主流程替换为显式 scope 驱动的 `validate` 工作流，并统一成功/失败输出策略。
* 重构 `libs/tracer_core/src/application/pipeline/**`，将旧 `context/manager/steps` 结构收口为 `pipeline_types.hpp`、`pipeline_stages.hpp` 与 `pipeline_orchestrator.hpp`，同步导出 `tracer.core.application.pipeline.*` canonical modules。
* 重构 `libs/tracer_core/src/application/**`、`libs/tracer_core/src/infrastructure/config/**`、`libs/tracer_core/src/infrastructure/serialization/json_serializer.cpp`、`libs/tracer_adapters_io/src/infrastructure/io/**` 与 `libs/tracer_transport/src/runtime_codec_*.cpp` 的项目内跨单元依赖，优先切换到现有叶子模块 `import`。
* 重构 `apps/tracer_core_shell/tests/**`、`libs/tracer_core/tests/**` 与 `libs/tracer_transport/tests/**` 的大型测试文件，按运行时回归、报表一致性、data query、transport codec 等职责拆分夹具与场景文件。
* 重构 `apps/tracer_cli/windows/rust_cli/src/cli/mod.rs` 与旧 `core/runtime/callbacks.rs`，分别按命令族和平铺终端回调职责拆分，保持原有 CLI 参数、回调桥接和日志语义不变。

### 修复 (Fixed)
* 修复验证工具成功日志过长、失败信息分散的问题，改为成功时极简输出，失败时自动提炼关键错误并写入 `out/validate/<run_name>/summary.json`。
* 修复 `workflow_handler`、`import_service`、`config loader`、`runtime codec` 等路径对隐式 include 链的依赖，补齐 `pipeline`、`tree viewer`、`diagnostics`、`string_utils` 等显式模块消费。
* 修复 `apps/tracer_core_shell/tests/**` 与 `libs/tracer_transport/tests/**` 中多类超大测试文件难以定位职责和回归来源的问题，拆分后保留原有验证范围并减少维护噪音。

### 弃用/删除 (Deprecated/Removed)
* 删除 `tools/toolchain/cli/handlers/post_change.py`、`tools/toolchain/commands/cmd_workflow/post_change*.py` 与 `tools/toolchain/services/change_policy/*`，不再保留旧 `post-change` 语义与状态文件链路。
* 删除 `libs/tracer_core/src/application/pipeline/context/pipeline_context.hpp`、`pipeline_manager.hpp/.cpp`、`steps/pipeline_stages.hpp/.cpp` 以及旧 `tracer.core.application.pipeline.context` / `manager` 模块接线，统一切到新的 pipeline family 结构。
* 删除 `libs/tracer_core/tests/application/tests/application_legacy_headers_compat_tests.cpp`，不再维持旧 application legacy header 兼容壳。
