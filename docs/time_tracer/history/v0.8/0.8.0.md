## [v0.8.0] - 2026-03-07

### 技术改进 / 重构 (Changed/Refactor)
* 重构 `libs/tracer_core/src/domain/**`、`libs/tracer_core/src/application/**` 与 `libs/tracer_core/src/infra/serialization/**` 的 JSON 边界，统一只在输出层、传输层、序列化层与适配层处理 `nlohmann/json`。
* 重构 processed JSON 校验链路，将结构校验从 `libs/tracer_core/src/domain/logic/validator/json/rules/` 迁移到 `libs/tracer_adapters_io/src/infra/io/processed_json_validation.*`。
* 重构 data query 语义输出链路，将内部 `bool semantic_json` 收口为显式 `DataQueryOutputMode`，避免把 `semantic_json` 继续当作 core 内部协议传播。
* 重构 `apps/tracer_core_shell/host/android_runtime_factory.cpp` 与 `libs/tracer_core/src/infra/persistence/importer/repository.*` 的 SQLite 装配方式，将 runtime bootstrap 与 write repository 解耦，改为只有在 validation 之后才允许进入写路径。
* 重构报表与查询读路径，引入 `libs/tracer_core/src/infra/reports/lazy_sqlite_report_query_service.*` 与 `lazy_sqlite_report_data_query_service.*`，统一在缺库时返回显式错误，而不是隐式建库。
* 重构 `apps/tracer_core_shell/host/android_runtime_factory.cpp` 的 runtime bootstrap 输出边界，移除 `output_root`、`output/db` 等目录的提前创建，调整为只有在真正写入、落盘或输出错误报告时才创建。
* 重构 `libs/tracer_core/src/infra/logging/file_error_report_writer.*` 的错误日志生命周期，由构造时即初始化日志文件，改为首次 `Append()` 时再创建 `errors-*.log` / `errors-latest.log`，避免无错时产生空日志文件。

### 修复 (Fixed)
* 修复 `libs/tracer_core/src/application/**` 与 `libs/tracer_core/src/domain/**` 中 JSON 依赖持续向内核扩散的问题，避免业务层继续直接暴露 `nlohmann::json`。
* 修复 `libs/tracer_core/src/infra/serialization/` 头文件级 JSON 暴露问题，消除公共接口对 `nlohmann/json_fwd.hpp` 的依赖。
* 修复 ingest 在结构校验或逻辑校验失败时仍可能提前创建 `.sqlite3`、`-wal`、`-shm` 或 `-journal` 的问题，明确只有通过 persistence gate 后才允许建库和写入。
* 修复 runtime bootstrap、previous-tail 读取与 report/query 读路径之间的持久化边界泄漏，避免 runtime create、读路径探测或读前检查隐式触发建库。
* 修复 `report-chart` 等读路径在数据库缺失时的错误传播顺序，保证参数校验错误优先返回，且缺库不会再被“自动建空库”掩盖。
* 修复 `validate-logic`、`validate-structure` 等只校验命令在无错误时仍会生成 `output/db`、`output/logs`，以及空的 `errors-*.log` / `errors-latest.log` 的问题；现在“仅检验且无错误”路径不再产生任何落盘副作用。
* 修复 `libs/tracer_core/src/infra/persistence/sqlite_database_health_checker.cpp` 对 DB 父目录的过早强依赖，允许目录在启动阶段或只校验阶段不存在，改为进入真正 ingest/write path 时再创建。

### 弃用 / 删除 (Deprecated/Removed)
* 删除 `libs/tracer_core/src/domain/logic/validator/json/rules/json_rules.hpp` 与 `libs/tracer_core/src/domain/logic/validator/json/rules/json_rules.cpp`，移除 domain 层旧 JSON 直通校验接口。
* 删除 `libs/tracer_core/src/infra/reports/shared/utils/config/config_utils.hpp` 与 `libs/tracer_core/src/infra/reports/shared/utils/config/config_utils.cpp`，移除未使用的旧配置加载口子。
* 删除 `libs/tracer_core/src/infra/serialization/core/log_codec.hpp` 与 `libs/tracer_core/src/infra/serialization/core/log_codec.cpp`，移除旧的内部 JSON codec 兼容债务。
* 删除 Windows CLI 侧“失败后清理新建空库文件”的临时兼容逻辑，最终语义统一收口到 core/runtime 持久化边界本身。
