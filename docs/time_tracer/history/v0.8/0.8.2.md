## [v0.8.2] - 2026-03-11

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/tracer_core/**` 与相关 runtime 装配链路，明确 `v0.8.2` 的主线目标是把 `libs` 侧 C++20 modules 从 family bridge 推进到 `module-only` 收口。
* 重构 `libs/tracer_core/src/infrastructure/logging/**` 与 `libs/tracer_core/src/application/workflow/**`，完成 `Phase 4 / Milestone 1` 的首批 `module-only` 收口，统一 `modules_on` 默认装配到 canonical family import 与 `*.module.cpp` 实现路径。
* 重构 `libs/tracer_core/src/infrastructure/query/data/stats/**`、`libs/tracer_core/src/infrastructure/persistence/**` 与 `libs/tracer_core/src/infrastructure/reports/**`，完成 `Phase 4 / Milestone 2` 的低耦合叶子 `module-only` 收口，覆盖 `query.data.stats`、`persistence.write`、`persistence.runtime`、`reports.export`、`reports.dto`、`reports.querying` 与 `reports.data_querying` family。
* 重构 `apps/tracer_core_shell/host/android_runtime_factory.cpp` 与 `libs/tracer_core/tests/**`，让 runtime 装配、modules smoke 和 legacy compat 验证默认对齐 `tracer.core.*` canonical modules，而不再依赖 `modules_on` 下的 header-world 实现承接。
* 重构 `temp/import_phase4*.md` 的里程碑与 family 记录，明确 `Phase 4` 退出条件、`modules_on` 默认验证矩阵、以及 `config`、`reporting/tree + use_cases` 的后续 defer 边界。

### 修复 (Fixed)
* 修复 `modules_on` 路径下 `logging`、`workflow`、`query.data.stats`、`persistence.write`、`persistence.runtime` 与 `reports.*` family 仍回退到 legacy `.cpp` 承接实现的问题，统一稳定到 `module-only` 验证路径。
* 修复 `apps/tracer_core_shell/cmake/ProjectOptions.cmake` 在 `TT_CPP20_MODULES_EFFECTIVE=ON` 下的 `ccache + named modules` 构建不稳定问题，避免 `.pcm` side-output 丢失导致的增量构建失败。
* 修复 `config` mixed TU 扩面前的 readiness 判定与 `reporting/tree + use_cases` 的收尾边界，明确剩余阻塞来自 consumer fanout 与 toolchain 限制，而不是 owning implementation 缺失。

### 弃用/删除 (Deprecated/Removed)
* 删除 `logging`、`workflow`、`query.data.stats`、`persistence.write`、`persistence.runtime`、`reports.export`、`reports.dto`、`reports.querying` 与 `reports.data_querying` 在 `modules_on` 路径对 legacy `.cpp` 的实现承接与双轨 source 选择。
* 删除上述已收口 family 在 `modules_on` 路径中的 `TT_ENABLE_CPP20_MODULES` / `!TT_ENABLE_CPP20_MODULES` 业务分支，统一切到单一 canonical module 装配。
