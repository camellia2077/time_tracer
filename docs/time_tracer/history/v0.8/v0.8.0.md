## [v0.8.0] - 2026-03-07

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/tracer_core/src/domain/**` 与 `libs/tracer_core/src/application/**` 的 JSON 边界，明确 `nlohmann/json` 只保留在 `transport`、`serialization`、`renderer`、`adapter` 与宿主边界。
* 重构处理后数据导入链路，将原始 processed JSON 的结构校验从 `libs/tracer_core/src/domain/logic/validator/json/rules/` 迁移到 `libs/tracer_adapters_io/src/infrastructure/io/processed_json_validation.*`。
* 重构 `libs/tracer_core/src/infrastructure/serialization/json_serializer.*`，将公共接口统一为 `typed model <-> JSON text`，不再在公共头文件暴露 `nlohmann::json`。
* 重构 data query 语义输出链路，将内部 `bool semantic_json` 传播改为显式 `DataQueryOutputMode`，使 `semantic_json` 明确成为 renderer 边界输出模式而不是 core 内部协议。
* 重构 AI 扩展预留边界，在 `apps/tracer_core_shell/cmake/CoreTargets.cmake` 中为未来 `libs/tracer_core_ai/src/domain/**` 与 `libs/tracer_core_ai/src/application/**` 建立同级 JSON 边界约束。
* 收紧 `C:\code\time_tracer\.agent\rules\AGENTS.md` 的 JSON 规则，要求 core 与未来 AI 模块优先处理 typed semantic model，仅在输出层、传输层和适配层进行 JSON 编解码。

### 修复 (Fixed)
* 修复 `libs/tracer_core/src/application/**` 与 `libs/tracer_core/src/domain/**` 中 JSON 依赖持续向内核扩散的问题，避免业务层继续把 `nlohmann::json` 作为输入输出类型。
* 修复 `semantic_json` 容易被误用为 application 内部协议的风险，统一收口到 query renderer 输出边界。
* 修复 `libs/tracer_core/src/infrastructure/serialization/` 中头文件级 JSON 暴露问题，消除公共接口对 `nlohmann/json_fwd.hpp` 的依赖。

### 弃用/删除 (Deprecated/Removed)
* 删除 `libs/tracer_core/src/domain/logic/validator/json/rules/json_rules.hpp` 与 `libs/tracer_core/src/domain/logic/validator/json/rules/json_rules.cpp`，移除 domain 层旧 JSON 直通校验接口。
* 删除 `libs/tracer_core/src/infrastructure/reports/shared/utils/config/config_utils.hpp` 与 `libs/tracer_core/src/infrastructure/reports/shared/utils/config/config_utils.cpp`，移除未使用的旧 JSON 配置加载口子。
* 删除 `libs/tracer_core/src/infrastructure/serialization/core/log_codec.hpp` 与 `libs/tracer_core/src/infrastructure/serialization/core/log_codec.cpp`，移除旧的内部 JSON codec 头接口与兼容债务。
