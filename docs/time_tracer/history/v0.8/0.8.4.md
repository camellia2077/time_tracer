## [v0.8.4] - 2026-03-21

### 新增功能 (Added)
* 新增 `tracer exchange` 事务式完整导入能力：导入流程固定为校验并应用包内 `converter TOML`、校验全部有效 TXT、覆盖包内月份、保留包外月份，并按新配置全量重建数据库。
* 新增 `ReplaceAllData` 与 `RunIngestReplacingAll(...)` 全量替换入库能力，为交换包事务导入提供单事务数据库重建边界。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/tracer_core/src/infra/crypto/**` 与 `tracer exchange` 请求/响应 DTO，删除旧的“解包到 work root / 可选 `apply_config`”导入语义，统一 `RunTracerExchangeImport` 为事务式完整导入入口。
* 重构 `tracer exchange` 包契约到 `package_version = 3`，将 payload 路径固定为 `payload/<year>/YYYY-MM.txt`，并要求路径年月、文件名与 TXT 内容头 `yYYYY + mMM` 完全一致。
* 重构 `apps/tracer_core_shell/api/c_api/**`、`apps/tracer_core_shell/api/android_jni/**` 与 `apps/tracer_core_shell/host/**` 的导入桥接，统一导入结果为“更新 active text root + 应用 converter config + 重建数据库”的事务式语义。
* 重构 `apps/cli/windows/rust/src/commands/handlers/crypto.rs` 与相关 CLI 测试，`crypto decrypt` 统一走事务式完整导入，不再沿用旧的解包输出目录心智。
* 重构 `libs/tracer_core/src/application/**` 的入口与编排边界：新增 `PipelineApi / QueryApi / ReportApi / TracerExchangeApi / TracerCoreRuntime` capability 聚合结构，将旧的“单一总入口 + 单一总编排”收敛为按能力分工的 application 入口。
* 重构 `libs/tracer_core/src/application/pipeline/**`：新增 `IPipelineWorkflow / PipelineWorkflow`，将 convert / ingest / import / validate 相关编排集中到 pipeline-only workflow 边界。
* 重构 `apps/tracer_core_shell/host/android_runtime_factory.cpp` 与 `apps/tracer_core_shell/api/c_api/**` 内部接线：宿主运行时新增 `runtime_api` 聚合入口，C API / JNI 内部改为通过 capability accessor 调用 `pipeline / query / report / tracer_exchange`。
* 重构 `libs/tracer_core/src/application/modules/**` 与 `cmake/sources/application_*`：application 入口层改为常规 `.hpp/.cpp` 实现，同时保留薄 module surface 作为导出包装，减少旧 `.module.cpp / .inc` 拼装路径对主实现的绑定。
* 重构 `libs/tracer_core/tests/application/**`：application test support 改为围绕 `FakePipelineWorkflow + BuildRuntimeApiForTest(...)` 组织，模块测试从旧 `BuildCoreApi* / FakeWorkflowHandler` 命名切换到 runtime aggregate 与 capability accessor 语义。
* 重构 `apps/tracer_core_shell/tests/platform/**` 的高频 Android runtime / file-crypto 用例：测试主路径改为依赖 `runtime_api`，不再优先通过兼容 `core_api` 字段访问能力。

### 修复 (Fixed)
* 修复 `tracer exchange` 导入仍可能在旧 active config 下逐月 ingest TXT，导致包内 TOML 与 TXT 解析/入库语义不一致的问题。
* 修复导入流程先落本地 TXT 再校验、失败后易留下半更新状态的问题，事务式导入现在在失败时统一回滚 config、TXT 与数据库。
* 修复数据库只支持追加或单月替换，无法在配置全局变化后按全部本地 TXT 重建数据的问题。

### 弃用/删除 (Deprecated/Removed)
* 删除旧 `tracer exchange` 导入中的 `requested_output_path`、`runtime_output_root` 与 `apply_config` 兼容 request 字段，以及 `resolved_work_root`、`applied_config_dir` 等旧结果模型。
* 删除 `tracer exchange` `v2` 导入兼容路径，旧 `v2` 包不再支持读取。
* 删除 `infrastructure::bootstrap::AndroidRuntime.core_api` 兼容字段；Android runtime host 统一通过 `runtime_api` 聚合入口暴露 capability 访问。
