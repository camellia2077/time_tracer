## [v0.10.0] - 2026-04-13

### 技术改进/重构 (Changed/Refactor)
* 统一 `libs/tracer_core`、`libs/tracer_transport` 与 `apps/tracer_core_shell` 的 reporting contract，只保留 `tracer_core_runtime_temporal_report_json` 作为 canonical 单次报告 ABI，并让 `query|structured_query|targets|export` 复用同一条 temporal 请求链路。
* 重构时间查询接口，改为以 `display_mode=day|week|month|year|range|recent` 配合 `selection_kind=single_day|date_range|recent_days` 表达查询意图，并在 temporal selection 中新增可选 `anchor_date`。
* 将 `week|month|year` 收敛到 `date_range` 选择模型，但在 core 内继续保留各自的 formatter 与 missing-target 语义，避免统一 contract 后退化成普通 `range` 行为。
* 在应用层解析 anchored recent，把 `recent + anchor_date` 规范化为固定窗口 `[anchor_date-(days-1), anchor_date]` 取数，再恢复 recent 的标题、命名与 `requested_days` 语义。

### 修复 (Fixed)
* 修复 reporting 查询接口长期并存导致的语义漂移问题，避免 `day`、`range`、`recent`、targets 与 export 分别走不同 payload 形状后出现宿主侧重复转换、重复校验与行为不一致。
* 修复 recent 固定窗口需求只能由宿主侧改写成 `range` 的问题，让 core 直接理解 anchored recent，减少 Android、CLI、Windows 等宿主各自处理当前时间与窗口换算的分叉逻辑。
* 修复 transport 与 C ABI 在 legacy request/payload 上长期保留兼容层带来的维护风险，统一 codec、能力标记与错误返回口径。

### 弃用/删除 (Deprecated/Removed)
* 删除 legacy reporting DTO 与 API，包括 `ReportQueryType`、`ReportExportType`、`ReportTargetType`、`ReportQueryRequest`、`StructuredReportQueryRequest`、`ReportTargetsRequest`、`ReportExportRequest` 及对应 helper / wrapper。
* 删除旧 reporting transport payload codec 与 C ABI 入口，包括 `tracer_core_runtime_report_json`、`tracer_core_runtime_report_targets_json`、`tracer_core_runtime_export_json` 的 reporting 用法与兼容桥接实现。
