## [v0.7.2] - 2026-03-05

### Added
* 新增 **C++20 Modules** 分阶段落地方案文档：`temp/module-refactor-partial.md` 与 `temp/core-module.md`，明确 `Phase 0~6` 迁移边界、门禁与回滚策略。
* 新增 modules smoke/legacy compat 测试矩阵，覆盖 **tracer_transport**、**tracer_adapters_io** 与 `libs/tracer_core/src/{shared,domain,application,infrastructure}` 模块面。
* 新增 Android 构建覆盖参数 `-PtimeTracerEnableCpp20Modules=ON|OFF`，并透传到 `TT_ENABLE_CPP20_MODULES`。

### Changed/Refactor
* 重构模块目录布局：根目录从 `modules/` 收敛到 `libs/`，各库内继续保留 `src/modules/*.cppm`。
* 重构 **tracer_transport** named modules 试点：聚合模块 `tracer.transport` 与子模块 `errors/envelope/fields/runtime_codec_*` 进入模块文件集。
* 重构 **tracer_adapters_io** 分层模块桥接：优先迁移 `file_reader`、`file_writer`、`file_system_helper`、`file_utils`，并保持旧头桥接兼容。
* 重构 **tracer_core** 模块主路径：完成 `shared`、`domain`、`application`、`infrastructure`（低耦合）模块迁移，统一命名为 `tracer.core.<layer>.*`。
* 重构 **tracer_core infrastructure 构建归属**：将 `reports_shared/reports_data/ttri/tti/time_tracker_infrastructure` 构建职责收敛到 `libs/tracer_core/infrastructure/CMakeLists.txt`，`apps/tracer_core/src/infrastructure/CMakeLists.txt` 收敛为壳层接线。
* 同步默认构建策略到 `Host ON / Android ON`，并在 `apps/tracer_core/CMakeLists.txt` 与 `apps/android/runtime/build.gradle.kts` 收敛同一开关语义。
* 重构跨层 include 热点：在 `libs/tracer_core/src/application/` 与 `libs/tracer_core/src/infrastructure/` 收敛前置声明与显式 include，降低头文件级联依赖。
* 固化 app 语义边界：`libs/tracer_core` 作为复用实现层，`apps/tracer_core` 保留对外入口语义（API/JNI/entry/packaging）。

### Fixed
* 修复 Android 路径下 SQLite 头传递依赖问题：新增 `libs/tracer_core/src/infrastructure/sqlite_fwd.hpp`，并在实现单元显式 `#include <sqlite3.h>`。
* 修复 module interface 重依赖暴露问题：将 JSON 前置声明统一到 `nlohmann/json_fwd.hpp`，降低 `nlohmann_json` 耦合。
* 修复兼容层冗余：删除纯转发头 `apps/tracer_core/src/infrastructure/config/models/converter_config_models.hpp`，并让 `converter_config_loader.hpp` 直接依赖 `domain/types/converter_config.hpp`。

### Deprecated/Removed
* 删除 `apps/tracer_core/src/infrastructure/config/models/converter_config_models.hpp` 纯转发兼容头，落实 module-first 准入规则。
* 删除 `apps/tracer_core/src/infrastructure/CMakeLists.txt` 中遗留 alias 依赖链，改为直接链接 `tracer_adapters_io` 与 `ttri`。
