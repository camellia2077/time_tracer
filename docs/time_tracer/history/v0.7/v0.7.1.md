## [v0.7.1] - 2026-03-02

### Changed/Refactor
* 删除旧插件链路文件：`formatter_c_*`、`formatter_abi_wrapper_template.hpp`、`formatter_config_payload*`、`report_data_payload_v1.hpp`、`dll_formatter_wrapper.hpp`，完成进程内 formatter 收口。
* 删除旧插件 wrapper 与子目录构建定义：`day|month|range` × `md|tex|typ` 的 `*_formatter.cpp` 与对应 `CMakeLists.txt`。
* 删除旧插件配置与校验模型：移除 `runtime_environment_requirements`、`startup_validator`、`plugin_validator` 相关路径，去除 `expected_formatter_plugins` 与 `plugins_directory` 语义。
* 增加 Core 契约基线与边界：固定 `ingest/query/export/tree/crypto` 命令样本并保留 `result.json`、关键日志与核心返回 JSON，约束契约聚焦状态码与结构化字段。
* 重构 Core 日志输出行为：`BuildAndroidRuntime` 默认使用 Null logger/sink，未注入日志端口时不再输出控制台。
* 增加 C ABI 回调接口：`set_log_callback`、`set_diagnostics_callback`，runtime 创建时绑定回调适配器，未注册时静默回落。
* 重构 Rust CLI 日志呈现链路：注册 Core 回调并统一 `stdout/stderr` 输出策略，Windows 启动时初始化 UTF-8 控制台编码。
* 重构失败返回契约：稳定输出 `error_code`、`error_category`、`hints`，测试口径改为契约字段断言。
* 完成 Phase 5 首轮：落地配置加载缓存与关键阶段耗时观测（解析、查询、导出）。

### Added
* 新增 `verify` 报告三格式快照门禁：`day/month/range` × `md/tex/typ` 固定样本 byte-audit，基线目录为 `test/golden/report_triplet/*/v1`。

### Deprecated/Removed
* 下线插件目录依赖叙述：构建与 workflow 文档统一切换为进程内 formatter 语义。

### Fixed
* 修复中文输出可读性风险：Rust CLI 在 Windows 控制台默认启用 UTF-8，降低乱码概率。

### Validation
* `python scripts/run.py post-change --app tracer_core --run-tests always --build-dir build_fast --concise` 通过（构建、`artifact_windows_cli`、native core smoke/stability）。
* `build` 与 `build_fast` 目录下关键命令路径一致性验证通过。
