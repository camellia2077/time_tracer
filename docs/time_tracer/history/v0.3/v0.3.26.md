## [v0.3.26.6] - 2026-01-10

### 技术改进/重构 (Changed/Refactor)
* **模块重命名与职责泛化**: 将原有的 `db_inserter` 模块重命名为 **`importer**`。它现在承担起从外部数据源（JSON/内存对象）到内部存储（SQLite）转化的完整业务职责。
* **Importer 与 IO 彻底解耦**:
    * **去 IO 化**: **`importer**` 模块不再包含文件读取代码，接口仅接收纯数据（如内存 JSON 字符串或 `InputData` 对象列表）。
    * **控制反转**: 文件读取职责上移至 **`Core**` 层（具体在 **`WorkflowHandler**` 中），利用 **`IO**` 模块读取文件后传递给 **`importer**` 处理。
* **支持内存直连**: 现在的流水线支持 `Converter` (内存) -> `Importer` (内存) -> `DB` 的直接导入，无需磁盘中间文件，大幅提升性能。

## [v0.3.26.0] - 2026-01-09

### 技术改进/重构 (Changed/Refactor)
* **路径管理全面升级**: 核心组件全面采用 C++17 `std::filesystem::path` 替代 `std::string`。
* **核心类适配**: **`AppConfig**`, **`ConfigLoader**`, **`FileController**` 的路径接口及成员变量均已更新为 `fs::path`。
* **工具类升级**: **`ConfigUtils**` 的 `load_json_config` 函数现接受 `const std::filesystem::path&` 参数。
* **报表模块适配**: **`Day**`, **`Month**`, **`Period**` 各报表配置类构造函数均已适配路径类型。

> [!IMPORTANT]
> **ABI 兼容性警告**: 由于 **`AppConfig**` 内存布局变更，所有插件 DLL 必须重新编译，否则将导致程序崩溃。
