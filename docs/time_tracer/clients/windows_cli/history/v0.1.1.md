## [v0.1.1] - 2026-02-22

### CLI 输出收敛
- 统一未知命令输出流到 `stderr`，并补充灰色提示，避免顺序错乱。
- 收敛 pipeline 失败总结，避免顶层 `Logic Error` 与 core 失败总结重复。
- `import` 移除前置提示，减少路径内中英混杂。
- `tree` 的 `GetHelp()` 保持单行摘要；`--help` 总览改为列表强制单行摘要。
- `--help` 命令列表列宽改为按最长命令动态对齐，提升可读性。
- Windows 控制台启用 `STDOUT/STDERR` 双通道 VT；bootstrap 错误前缀统一为 `Runtime check failed: ...`。
- 验证：`python scripts/verify.py --app time_tracer --quick` 通过（`124` 通过，`0` 失败）。

### 测试套件命名统一
- Windows CLI 测试结果目录统一为 `test/output/tracer_windows_cli/`，不再使用 `test/output/time_tracer/`。
- 文档与脚本同步：`tracer_windows_cli` 明确为 `core + windows cli` 集成套件。
- `verify --app time_tracer` 入口保持不变，内部映射到 `tracer_windows_cli` 套件并构建 `apps/tracer_windows_cli`。

### Root 过滤语义收口
- `query --help` 中 `query data` 参数说明明确推荐 `--root` 作为根节点过滤主入口。
- `--project` 保留为兼容语义，并在帮助文案中标记为 legacy contains 过滤。
- `query data report-chart` / `query data days-stats` 的 `--root` 场景补充回归：
  - 合法 root
  - 缺失 root（无匹配）
  - 空 root（等价于不加 root）
  - `--root` 与 `--project` 同时传入时 `--root` 优先
- 验证：`python scripts/verify.py --app time_tracer --quick` 通过（`Total: 130, Failed: 0`）。
