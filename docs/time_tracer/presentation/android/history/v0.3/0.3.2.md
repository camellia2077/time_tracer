## [v0.3.2] - 2026-04-08

### 新增功能 (Added)
* 新增 Android 单个 `TRACER` 导入前确认对话框，在密码校验通过后先显示包创建时间、包版本、producer、`source_root_name` 与 payload 数量，再由用户确认后执行正式导入。
* 新增 `Config` 页 `Charts` 分类，使 `assets/tracer_core/config/charts/*.toml` 可以和 `converter`、`reports` 一样在 Android 端直接打开和编辑。
* 新增 `Config` 页 `Meta` 分类，使 `config.toml` 与 `meta/*.toml` 不再混在 `Converter` 里，而是作为低频修改的运行时元配置独立展示。
* 新增 Android `Config` 页 `Converter` 内的 `Aliases | Rules` 二级选择，其中 `Aliases` 专门面向 `converter/aliases/` 下的高频活动名转换 TOML，`Rules` 面向 converter 的稳定规则配置集合。
* 新增 `Converter > Aliases` 结构化编辑 MVP：默认进入 `Structured` 模式，支持 `parent`、分组与 alias 条目的树形增删改；保留 `Advanced` 模式用于直接编辑原始 TOML 文本。
* 新增别名子文件编解码层 `apps/android/app/src/main/java/com/example/tracer/data/AliasTomlEditorCodec.kt`，把 `converter/aliases/*.toml` 映射为内存树模型并支持稳定序列化回 TOML。
* 新增 `parent` 候选下拉选择（替代手动输入），候选来自当前活跃 alias 文件集合并按字母排序，降低 Android 端输入负担。
* 新增 alias 编辑相关多语言文案（`values/`、`values-zh/`、`values-ja/`）与单元测试 `AliasTomlEditorCodecTest`、`ConfigViewModelTest`。

### 技术改进/重构 (Changed/Refactor)
* 重构 Android 单个 `TRACER` 导入交互为 `select -> passphrase -> inspect -> confirm -> import`，设计意图是先暴露备份包身份信息，降低误导入旧 `data.tracer` 的风险，再执行会替换本地 TXT 并重建数据库的导入动作。
* 重构 Android tracer exchange inspect 结果模型，补充 `packageVersion` 到 Kotlin/UI 层，使导入前确认对话框可以直接展示当前包契约版本。
* 重构 Android TXT editor 的 day-block 职责边界：移除 UI 层本地 `MMDD` 默认值推导、day-block 解析与替换逻辑，改由 `RuntimeTxtDayBlockService` 通过 shared TXT runtime family 调用 `default_day_marker`、`resolve_day_block`、`replace_day_block`；Android 现在作为 runtime consumer，仅保留界面状态、输入路由与保存触发编排。
* 重构 Android `Config` 页文件分类逻辑，把 `charts/` 从原先隐式混在 converter 组里的 TOML 文件提升为独立分类，并把 `config.toml` / `meta/*.toml` 提升为 `Meta` 分类；设计意图是把高频修改项与低频运行时元配置分开，优化配置入口的信息架构与可发现性。
* 重构 Android `Config` 页 TOML 列表模型为“真实路径 + 展示名”双字段结构：运行时与保存链路继续使用 `converter/...`、`reports/...`、`charts/...` 这类 canonical relative path，UI 列表与编辑器标题改为展示去掉当前分类前缀后的名称；设计意图是把配置定位身份与分类内展示文本解耦，避免把 `Converter` / `Charts` / `Reports` 已经表达过的信息重复显示到每个条目上。
* 重构 Android `Config` 页 `Converter` 组的内部导航为 `Aliases | Rules`：`Aliases` 被定义为用户维护的 `converter/aliases/` 动态目录，而不是固定文件枚举；`Rules` 被定义为 converter 自带的稳定规则/索引 TOML 集合。设计意图是把高频别名维护入口提升为一级可见选择，同时避免把用户可扩展的 alias 文件集合错误建模成写死列表。
* 重构 Android `Config` 页 `Converter > Aliases` 的显示名生成逻辑：真实路径与稳定展示名保持不变，仍保留 `converter/aliases/meal.toml` 与 `aliases/meal.toml` 这层语义；只有在 `Config` 页渲染 `Converter > Aliases` 当前作用域时，才额外裁掉 `aliases/`，显示为 `meal.toml`。设计意图是只移除当前层级已经表达过的冗余上下文，不改动底层读写路径或跨层稳定标识。
* 重构 Android `Config` 编辑状态机与路由：在 `ConfigViewModel`、`ConfigScreen`、`TracerTabs` 中引入 alias 专用状态（`aliasDocumentDraft`、`aliasAdvancedTomlDraft`、`aliasParentOptions`、`aliasEditorMode`），使 `converter/aliases/*.toml` 与其他 TOML 文件走分流编辑链路。
* 重构 alias 保存前校验流程：在结构化/高级模式保存时统一执行 `parent` 与节点结构校验，并对当前活跃 alias 文件集合做 alias key 全局唯一性校验，再走 `saveConfigTomlFile(...)` 落盘。
* 重构高级模式切换语义：从 `Structured` 切换到 `Advanced` 时基于当前结构化草稿生成 TOML；从 `Advanced` 回 `Structured` 时要求 TOML 可解析，否则保留在高级模式并展示错误。
* 重构 Android `Converter > Aliases > Structured` 交互为单层 drill-down 导航：移除递归缩进卡片，改为“当前层分组/条目平铺 + 进入子分组”；设计意图是避免深层级在手机窄屏上出现“层级越深、卡片越窄”的可读性退化。
* 重构 Android alias 层级导航信息架构：新增路径栏与返回上一级交互，根层显示 `Current path: aliases`，子层显示完整路径；设计意图是把深度语义从横向缩进迁移到纵向导航，降低多层配置编辑的认知成本。
* 重构 Android `Converter > Aliases` 当前层操作语义：`Add Group` 与 `Add Alias` 固定作用于当前导航层，而非同屏展开层级内随机插入；设计意图是提升编辑操作的可预测性与深层级维护效率。
* 重构 Android alias 列表窄屏展示策略：分组标题与操作区拆为两行、操作以图标为主、路径栏支持换行且保持根层/子层一致高度、映射值采用 `→ canonicalLeaf` 展示；设计意图是在保留信息密度的前提下提升窄屏稳定性和映射可读性。

### 修复 (Fixed)
* 修复 Android 导入流程在密码通过后直接执行正式导入、难以及时发现旧备份包的问题，改为先显示关键摘要再确认导入。
* 修复 Android `Config` 页未暴露 `charts` 类配置、且把 `config.toml` 与 `meta/*.toml` 混在 `Converter` 中的问题，避免高频配置与低频元配置入口混杂。
* 修复 Android `Config` 页在 `Converter`、`Charts`、`Reports` 分类下重复显示 `converter/`、`charts/`、`reports/` 前缀的问题，分类内条目现在分别显示为 `interval_processor_config.toml`、`heatmap.toml`、`markdown/day.toml` 这类更短的名称，同时保持底层读写路径不变。
* 修复 Android `Config` 页 `Converter` 内 `Aliases | Rules` 仅显示分段按钮但未实际限制文件范围的问题，二级选择现在会真正按 `converter/aliases/` 与其余 converter 规则配置拆分可见 TOML 列表。
* 修复 alias 解析中对 `TomlTable.get(key)` 的 dotted-key 误解释问题，改为遍历 `entrySet()` 读取即时键，避免中文等 quoted alias key 被误判或解析失败。
* 修复 `Converter > Aliases` 场景仍需手动输入 `parent` 的交互问题，改为下拉选择并按字母排序候选，减少手机输入和拼写错误风险。
* 修复 Android alias 深层级场景（如 `aliases.math.calculus.definite-integral`）在树形递归布局下的横向挤压问题，避免长分组名和映射值因层级过深而被过度截断或形成狭窄展示条。
