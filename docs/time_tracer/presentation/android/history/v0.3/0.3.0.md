## [v0.3.0] - 2026-04-02

* 说明：本文中的 `authorable` 校验，指“某个原始活动 token 是否允许被用户在 Android `Record Input`、`Quick Access` 或 TXT 作者态直接写入文本”；当前口径为 `alias_mapping.keys ∪ wake_keywords`，不等同于最终 ingest 后的 canonical 活动映射结果。

### 新增功能 (Added)
* 新增 Android report 结果的结构化同步能力，在 `apps/android/contract` 引入 `ReportErrorContract` 与 `ReportWindowMetadata`，用于承接 libs reporting runtime payload 中的 `error_contract` 与 `report_window_metadata`。
* 新增 Android `Report` 结果区摘要卡，在 `day/week/month/year` 目标不存在时展示明确的缺失摘要，并在 `recent/range` 返回空窗口时展示窗口范围、请求天数、匹配天数与匹配记录数。
* 新增 Android 侧 `listActivityAliasKeys()`、`listWakeKeywords()` 与 `listAuthorableEventTokens()` 查询能力，用于分别拉取 `alias_mapping.toml` 左键集合、`interval_processor_config.toml` 中的 `wake_keywords` 集合，以及作者态可输入 token 并集。
* 新增 `Record Input` 的显式时间排序模式透传能力，在 Android 合约与 runtime 链路引入 `RecordTimeOrderMode`，并支持 `strict_calendar` 与 `logical_day_0600` 两种模式。
* 新增 `Record Input` 的 `Record to Yesterday/Today` 选中态高亮样式，选中项使用 `primaryContainer` 填充并提升文本字重，增强当前目标日的可视化区分。
* 新增 `Quick Access` 搜索输入的显式 `Add` 入口与键盘 `Done` 提交流程，并在无匹配项时展示提示文案，用于解决“只能筛选看候选但缺少明确添加动作”的可用性问题。

### 技术改进/重构 (Changed/Refactor)
* 重构 Android report runtime 解析链路，扩展 `apps/android/runtime` 的 `NativeResponseCodec`、`NativeReportTranslator` 与 `ReportCallResult`，统一透传 `error_code`、`error_category`、`hints` 与 report window metadata，同时保持稳定 envelope `ok/error_message/content` 不变。
* 重构 `apps/android/feature-report` 的 report 状态模型与结果缓存，改为按 period 同时保存 markdown 与摘要语义，确保切换 `day/week/month/year/recent/range` 时能够稳定恢复各自的 report 摘要与结果内容。
* 重构 Android `Report` 文案提供器与多语言字符串资源，在 `values`、`values-zh`、`values-ja` 中补齐目标缺失、空窗口与窗口元数据展示文案，统一 report 侧新语义的用户可见反馈。
* 重构 `apps/android/runtime` 的 `Record Activity` 提交流程，改为通过 libs 级原子接口执行活动名匹配、候选 TXT 校验与正式提交，避免 Android 侧继续走“先写正式 TXT 再 ingest”的旧链路。
* 重构 Android 侧 `Record Activity` 的目标日决策，统一由 `RecordUseCases` 按显式策略分派：`YESTERDAY -> logical_day_0600`，`TODAY -> strict_calendar`。
* 重构 Android 侧目标日状态管理为“会话级共享”，移除离开 `Record` tab 即清空 override 的行为，并保持 `Record/TXT` 两个 tab 共享同一目标日意图。
* 抽取统一逻辑日解析器 `apps/android/feature-record/src/main/java/com/example/tracer/ui/viewmodel/RecordLogicalDayResolver.kt`，集中处理 `06:00` 切日与 `yesterday/today` 日期解析。
* 抽取分段数字输入能力到共享 UI 模块 `feature-ui-common`，并在 `TXT Editor` 的日期输入中复用，确保 `MM/DD` 录入交互与校验规则在多处界面保持一致。
* 重构 `TXT Editor` 默认 `MMDD` 计算路径，改为复用统一 resolver，并按 `Record to Yesterday/Today` 联动选择“实际日期/前一天”。
* 重构 `RecordTabContent` 到 `RecordScreen` 的 quick-activities 更新回调签名为 `Boolean` 返回值，仅在保存成功时才清空搜索输入与收起键盘，用于避免校验失败时误清空用户输入并提升反馈一致性。
* 重构 `TracerTabs`、`RecordTabContent`、Quick Access 搜索候选与保存校验的数据源，改为统一使用 `authorable_event_tokens = alias_mapping.keys ∪ wake_keywords`，解决作者态“普通活动映射集合”和“wake 语义集合”长期混用造成的口径歧义。
* 重构 `Quick Access` 快捷活动布局与排序交互，改为按文本长度自适应按钮宽度的流式排布，并继续支持长按进入自由拖拽排序，用于避免短活动名按钮被固定等宽网格拉得过宽，同时保留现有排序能力与触觉反馈。
* 抽取共享输入框样式 `TracerOutlinedTextFieldDefaults.shape`，并将 `Report` 侧全部输入框圆角统一为与 `Record Input` 一致的 `MaterialTheme.shapes.large`，用于避免不同报表参数输入在同一页面出现圆角口径不一致的问题。
* 抽取共享分段选择样式 `TracerSegmentedButtonDefaults`，并将 `Report` 与 `Config` 中的 `SegmentedButton` 选中态统一为与 `Record` 一致的高亮容器色、边框色与字重反馈，用于避免配置页和报表页在切换选项后缺少明确选中态颜色提示的问题。
* 重构 Android 本地 completeness warning 扫描与 wake 判断链路，移除 `isWakeLikeActivity(...)` 硬编码，实现为仅根据 core/runtime 返回的 `wake_keywords` 判定“首条 authored event 是否 wake”。
* 重构 `Record Time` 目标日切换文案与布局：将公共前缀提升为独立标题（`Record to`/`记录到`/`記録先`），按钮仅保留短文案（`Yesterday`/`Today`、`昨天`/`今天`、`昨日`/`今日`），并对按钮文本启用 `maxLines = 1 + TextOverflow.Ellipsis` 与等宽布局，用于在不同设备宽度下稳定保持单行与等高显示。

### 修复 (Fixed)
* 修复 Android `Report` 对 libs reporting 新语义的丢失问题，避免 `day/week/month/year` 的 `reporting.target.not_found` 在 Kotlin 层退化为普通失败文本。
* 修复 Android `recent/range` 空窗口结果缺少显式边界信息的问题，改为保留成功 markdown 输出，并同步展示 `has_records`、匹配天数、匹配记录数与请求窗口摘要。
* 修复 Android `Report` 结果切换时摘要语义无法随 period 一起恢复的问题，避免用户在不同报表模式间切换后看到错误的空状态或缺失提示。
* 修复 `Record Input` 与 `Quick Access` 无法录入仅存在于 `interval_processor_config.toml -> wake_keywords`、但不在 `alias_mapping.toml` 左键中的 wake 词的问题，解决了“配置上合法的起床 token，Android 作者态却无法输入”的业务断裂。
* 修复 Android 本地 wake 识别与 core ingest 口径可能不一致的问题，避免因本地硬编码字符串或 alias-only 校验导致通宵 warning、首条 wake 判断与实际入库语义偏离。
* 修复 `Record Input` 用户体验：为活动名录入接入单行作者态 token 匹配校验，并在每次 `Record Activity` 调用返回后统一清空 `Activity Name` 与 `Remark (Optional)` 输入框，避免用户下一次录入前还要手动删除上一次输入的内容。
* 修复 `Quick Activities` 交互限制，允许快捷活动列表完全为空，便于用户在导入新配置后先清空与当前配置不匹配的默认活动名。
* 修复 `Quick Activities` 删除路径误复用新增校验的问题，删除活动名时不再要求通过活动名匹配校验，确保历史默认项或旧配置项可以被直接移除。
* 修复 `TXT Editor` 默认目标日与 `Record Input` 目标日不一致的问题，确保切换 `Record to Yesterday/Today` 后 `TXT` 侧自动填充日期与录入目标日语义一致。
* 修复 `Record/TXT` 跨 tab 切换时目标日意图丢失的问题，避免用户刚切换为 `yesterday` 后进入 `TXT` 又被重置为默认值。
* 修复与补强 Android 侧相关实现注释，明确“会话共享、不本地持久化、统一 resolver、不在 tab `onLeave` 清 override”的设计意图。
* 修复快捷活动默认值回填策略：仅在“首次未配置”时使用内置默认项，用户显式清空后保持为空，并补充 `UserPreferencesRepositoryTest` 覆盖该行为，用于防止“清空后又被自动恢复默认项”。
* 修复 Quick Access、建议活动名与单行录入提示文案，把“必须匹配 `alias_mapping.toml` 左键”的旧说法调整为“必须匹配配置中的可输入 token（alias key 或 wake keyword）”，用于对齐最终作者态业务口径。
