## [v0.2.3] - 2026-03-07

### 技术改进/重构 (Changed/Refactor)
* 重构 Android 许可证装载链路：改为显式打包并读取 `assets/aboutlibraries/aboutlibraries.json`，不再依赖 release 资源裁剪后仍保留 `raw` 元数据。
* 重构 Android TXT 种子数据打包策略：`debug` 变体继续生成并打包 `assets/tracer_core/input/full/*.txt`，`release` 变体不再携带测试 TXT。
* 增加 APK 级资产校验：构建阶段校验许可证资产存在，且 `release` APK 不包含 `input/full` 测试数据。
* 重构 Android 图标设计链路：以 `design/branding/master/time_tracer_brand_master_symbol.svg` 作为无背景品牌母版，并新增 `design/branding/platform/android/time_tracer_android_launcher_reference.svg` 作为 Android 平台安全区参考稿。
* 重构 Android launcher 资源为标准 adaptive icon 分层：`background / foreground / monochrome` 分离，前景与单色图标只承载品牌符号，背景层改为 full-bleed 白色底色，最终外轮廓交由 launcher mask 决定。
* 重构 Android About 版本展示链路：Core 版本改为从 `libs/tracer_core/src/shared/types/version.hpp` 自动解析并注入 `BuildConfig.TRACER_CORE_VERSION`，不再在 UI 层手填。
* 重构 Android app 版本来源：新增 `apps/android/meta/version.properties` 作为 `VERSION_CODE` / `VERSION_NAME` 的单一来源，`app/build.gradle.kts` 不再手填表现层版本。
* 同步 Android 文档与规则：`apps/android/README.md`、`apps/android/agent.md`、`docs/time_tracer/clients/android_ui/README.md`、`docs/time_tracer/clients/android_ui/specs/STRUCTURE.md` 明确版本单一来源与图标平台适配规则。

### 修复 (Fixed)
* 修复 `Open Source Licenses` 在 release 包与真机环境下偶发显示不可用的问题。
* 修复 Android 发布包误携带 `test/data` TXT 测试数据的问题。
* 修复 Android Config About 页仍显示过期 Core 版本 `0.6.2` 的问题。
* 修复 Android 图标直接套用统一导出版导致在部分国产桌面上视觉过大、箭头过于贴边的问题。
* 修复 Android 图标在部分小米 launcher 下出现白色底板外再包一层深色圆角边的黑边问题。
