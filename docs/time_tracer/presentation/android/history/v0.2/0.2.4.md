## [v0.2.4] - 2026-03-21

### 新增功能 (Added)
* 新增 Android 单文件 `TRACER` 导入的事务式完整交换包语义：导入时直接调用 runtime 完成 `converter TOML` 应用、TXT 校验、本地月文件替换与数据库重建。

### 技术改进/重构 (Changed/Refactor)
* 重构 Android `TracerExchangeGateway`、runtime bridge 与导入结果模型，导入结果改为返回 `replacedMonthCount`、`preservedMonthCount`、`rebuiltMonthCount`、`databaseRebuilt` 与 `retainedFailureRoot` 等事务结果字段。
* 重构 `Data` 页单文件 `TRACER` 导入流程，删除“先解包 payload 再逐个 `single_txt_replace_month` ingest”的旧 UI 编排，统一改为一次 runtime 事务导入。
* 重构 Android 导入进度消费链路，允许消费来自 core 事务导入阶段的自定义 progress phase，而不再假定导入进度只能映射到 file-crypto 的固定阶段枚举。

### 修复 (Fixed)
* 修复 Android 单文件 `TRACER` 导入仍在本机旧 config 下逐个 ingest TXT，导致包内 TOML 与导入数据解析语义不一致的问题。
* 修复 Android 导入过程中可能先改写本地 TXT、后校验失败的问题，避免留下半更新的 month TXT 状态。
* 修复 Android 导入完成后只替换包内月份入库、未按新 config 重建全部本地数据的问题。

### 弃用/删除 (Deprecated/Removed)
* 删除 Android 导入流程中旧的 payload 解包后逐个 ingest 辅助逻辑与相关无用导入失败文案，不再保留“bundle 解包成功但导入仍是 UI 侧二段式执行”的兼容路径。
