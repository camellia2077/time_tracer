## [v0.2.7] - 2026-03-26

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/android/runtime/src/main/cpp/CMakeLists.txt` 的 native bridge 源文件装配路径，改为对齐 `apps/tracer_core_shell/api/c_api/{runtime,capabilities/**}` 与 `apps/tracer_core_shell/host/{bootstrap,exchange}` 的新分层目录，不再继续依赖旧平铺路径。
* 重构 `apps/android/runtime/src/main/java/com/example/tracer/runtime/{NativeRuntimeBridge,NativeRuntimeController}.kt`，把 `validate_structure`、`validate_logic`、单报告生成与 crypto/tracer-exchange 进度监听统一收口到 `NativeRuntimeBridge`，明确 Kotlin runtime 对 raw JNI surface 的唯一适配入口。
* 重构 `apps/android/runtime/src/main/java/com/example/tracer/runtime/controller/{RuntimeRecordDelegate,RuntimeReportDelegate}.kt` 与 `IngestSyncUseCase.kt`，改为通过注入的 native action 调用 runtime bridge，降低 delegate/use case 对 `NativeBridge` 直接静态依赖。
* 更新 `apps/android/runtime/src/main/java/com/example/tracer/bridge/NativeBridge.kt` 的桥接注释，并同步补齐 `apps/android/runtime/src/test/java/com/example/tracer/RuntimeRecordDelegateTest.kt` 的依赖注入测试桩，使 Kotlin/JNI 边界文档和测试入口与新的 bridge 分层保持一致。

### 修复 (Fixed)
* 修复上一轮 core/shell C API 路径重构后 Android runtime `CMakeLists.txt` 仍引用旧 `host` 与 `c_api` 平铺源文件的问题，使 `python tools/run.py build --app tracer_android --profile android_style` 能重新找到 capability/runtime/bootstrap/exchange 对应的 native bridge 实现。
* 修复 Android Kotlin 层在 `record`、`report` 与 `auto-sync ingest` 流程里直接调用 raw `NativeBridge` 的边界不一致问题，统一由 `NativeRuntimeBridge` 转译 JNI 签名，减少后续 native contract 变动时的同步漏改风险。
