## [v0.2.5] - 2026-03-21

### 新增功能 (Added)
* 新增 `TracerScreenTransferCoordinator` 与共享 transfer callback/request/result 模型，统一 Android 单文件 `TXT` 导入、单文件 `TRACER` 导入与整库 `TRACER` 导出的选取、口令、进度与收尾编排。
* 新增 `RuntimeCoreAdapter` 与 `RuntimeDiagnosticsService` 以及对应单元测试，将 native init/query/tree 调用与 diagnostics payload/list 组装从 `NativeRuntimeController` 中拆成可独立验证的服务边界。

### 技术改进/重构 (Changed/Refactor)
* 重构 `TracerScreen` 入口组织方式，拆出 `TracerScreenActions`、`TracerScreenContent`、`SyncTracerScreenRecordPreferences`、`TracerScreenReportHeatmapState`、`TracerScreenImportWorkRoot` 等协作单元，移除旧的超长内联 actions/imports/heatmap side effect 代码块。
* 重构 Android capability 注入边界：`MainActivity`、`AppContainer` 与 `TracerScreen` 不再把宽口径 `RuntimeGateway` 直接下发到各 tab，改为按职责显式注入 `QueryGateway`、`ConfigGateway`、`TracerExchangeGateway` 等接口；`ConfigViewModel` 与 tab lifecycle 也同步切到 capability 级依赖。
* 重构 `TracerScreenExports`、单文件 `TXT` 导入与单文件 `TRACER` 导入流程，统一走共享 transfer coordinator，收敛原先分散在导入/导出分支里的选择取消、失败格式化、passphrase 输入与 finally 清理逻辑。
* 重构 `NativeRuntimeController` 内部接线：保留 `RuntimeGateway` 聚合对外契约，但将 native core call、diagnostics、query/report/config 等职责进一步转交独立 adapter/service 协作对象。
* 同步 Android 活动文档分层：补齐 `docs/time_tracer/presentation/android/history/README.md`、`docs/time_tracer/presentation/android/specs/DOC_RULES.md` 与 `reference/*.md`，将导航、行为参考与历史资料拆到独立层级。

### 修复 (Fixed)
* 修复 Android 单文件 `TXT` 导入仍使用硬编码英文状态文案的问题，`values`、`values-zh` 与 `values-ja` 现已补齐选择、取消与读取失败相关文案。
* 修复 Android 导入/导出链路在取消或异常时状态收口不一致的问题，统一由 transfer coordinator 负责清理进度状态、回写 status text，并在成功后继续同步 `Record` 页历史列表。

### 弃用/删除 (Deprecated/Removed)
* 删除旧的 `TracerScreenImports.kt` 平铺导入实现与 `TracerScreenCoordinator.kt` 旧协调入口，改由按职责拆分的 transfer/import helper 文件承载主线逻辑。
* 删除 `TracerScreen.kt` 中旧的内联 diagnostics copy、record preference sync 与 report heatmap auto-apply 直接实现，避免主入口继续堆积跨 tab 的副作用细节。
