## [v0.4.1] - 2026-04-16

### 新增功能 (Added)
* 新增 Android report `Breakdown` 图表语义，正式将 chart 结果区拆分为 `Trend` 与 `Breakdown` 两种视图，并为 breakdown 增加 `Pie` 与 `Horizontal Bar` 两种可视化。
* 新增 `DAY` 的 breakdown horizontal bar 展示，让单日 root 时长对比可以直接按排序和占比查看，不再只能依赖 pie。
* 新增 `report_composition` Android query/runtime 链路与对应 contract model，使 report 页面可以直接请求 breakdown 聚合数据而不是复用 trend 的 daily series。
* 新增 `DAY` composition `Treemap` 视图，复用现有 breakdown 聚合 payload 展示单日 root 时长面积分布，并保留下方 detail 区承接完整 root、时长与占比信息。
* 新增 Config `Breakdown Chart Palette` 预设持久化与实时预览，使 pie 与 horizontal bar 可以共享用户选定的 breakdown 配色方案。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/android/feature-report` 的 chart 状态模型，分离 `Trend` 与 `Breakdown` 的 render state、visual selector 与数据加载逻辑，避免不同图表语义继续共享同一套状态。
* 重构 report chart 文案，统一使用 `Trend / Breakdown`、`趋势 / 分布`、`推移 / 内訳`，并将 `Vertical Bar` 与 `Horizontal Bar` 明确区分，减少两类 bar 图的语义混淆。
* 重构 Config 中的 report chart 配色设置，将原先仅面向 `Pie` 的 palette 提升为 `Breakdown Chart Palette / 分布图配色 / 内訳チャート配色`，并在预览中同时展示 pie 与 horizontal bar。
* 重构 `apps/android/app/src/main/java/com/example/tracer/ui/screen/ConfigAppearanceSettingsCard.kt`，将 appearance 主编排、`Report Chart Style` 子域与通用展开按钮按职责拆分到独立文件，降低 Config 配色设置继续扩展时的耦合。
* 重构 `apps/android/app/src/main/java/com/example/tracer/ui/viewmodel/ConfigViewModel.kt` 与 `apps/android/app/src/main/java/com/example/tracer/ui/screen/ConfigAliasEditorCard.kt`，保留单入口 ViewModel / card 的同时，将 browsing、alias editing helper，以及 rows/selectors、dialogs 分离到独立文件，减少 Config 模块内部职责混杂。
* 调整 `apps/android/gradle.properties` 与 Android Gradle 默认参数，收紧并行和 configuration cache 行为，以降低 Windows 上 `built_in_kotlinc` 中间产物锁与状态复用问题。

### 修复 (Fixed)
* 修复 report 参数切换后 chart 仍可能残留旧结果的问题；现在时间窗口变化会失效旧 chart，并在 chart 模式下按当前语义自动重查。
* 修复 `DAY` 模式下 pie 只显示部分 root 的问题；单日 breakdown 现在保留完整 root 列表，而多日窗口继续使用更适合 pie 的聚合口径。
* 修复多日 breakdown pie 使用错误数据语义的问题，确保 `week|month|year|range|recent` 的 pie 与 horizontal bar 都基于当前窗口的 root 总时长聚合结果。
* 修复 breakdown 图在浅色/深色主题下颜色可读性不稳定的问题，改为使用稳定的 preset palette，并让 pie 与 horizontal bar 共用同一套 root 颜色映射。
* 修复 Config `Report Chart Style` 展开区把完整 preview 与预设列表混进按钮胶囊容器的问题，改为折叠态仅显示 swatch 摘要、展开区单独承载完整 preview，并补上对应 UI 回归测试。
* 修复 `DAY` treemap 在窄块内被动裁切横排文本的问题，改为按 tile 尺寸分级显示名称与摘要，并在空间不足时主动省略或交由共享 detail 区显示完整信息。
