## [v0.4.0] - 2026-04-13

### 新增功能 (Added)
* 新增 Android temporal reporting contract，使用 `TemporalReportQueryRequest` / `TemporalSelectionPayload` 统一表达 `day|week|month|year|range|recent` 报告请求，并预留 `anchorDate` 支持 anchored recent。
* 新增 Week 参数组件与时间选择解析链路，包括 `ReportTemporalInputComponents.kt`、`ReportWeekPickerState.kt` 与 `ReportTemporalSelection.kt`，让 report text 与 report chart 复用同一套时间选择逻辑。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/android/contract` 与 `apps/android/runtime` 的 reporting 入口，只保留 `ReportGateway.reportMarkdown(request)`、`nativeReportJson(requestJson)` 与 temporal JSON codec，不再按 day/month/week/recent/range 维护多组 wrapper 方法。
* 重构 Android report chart 参数解析，改为直接复用 report 当前的 temporal selection，而不是再维护独立的 chart 日期输入模式与单独的 lookback/range 状态。
* 重构 report 页面参数区与结果区联动，使 `TEXT` 与 `CHART` 在同一套 report 参数模型上切换，并让 chart 模式直接消费当前 report mode 的时间窗口。

### 修复 (Fixed)
* 修复 Android reporting 仍依赖 legacy `nativeReport(...)` JNI 签名的问题，统一到 `nativeReportJson(...)` 后避免 Kotlin / JNI / C ABI 三层 reporting 入口继续分叉。
* 修复 chart 与 text 使用不同时间参数模型导致的行为漂移问题，确保 `week|month|year|range|recent|day` 在 chart 查询时与当前 report 参数保持一致。
* 修复 report 参数变更后 chart 仍可能复用旧结果的问题；现在 `reportMode`、日期、week、month、year、range、recent 天数变化都会失效旧 chart，并在当前处于 chart 模式时自动重查。

### 弃用/删除 (Deprecated/Removed)
* 删除 Android legacy reporting wrapper 方法，包括 `reportDayMarkdown`、`reportMonthMarkdown`、`reportYearMarkdown`、`reportWeekMarkdown`、`reportRecentMarkdown` 与 `reportRange(...)`。
* 删除 raw JNI `nativeReport(...)` 及其 bridge 适配代码，并移除 Android runtime CMake 中对 legacy reporting source 的引用。
