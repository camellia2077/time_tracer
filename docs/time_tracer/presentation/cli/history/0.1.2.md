## [v0.1.2] - 2026-03-28

### 技术改进/重构 (Changed/Refactor)
* 重构 Windows CLI `report export` 职责边界，改由 CLI 统一负责导出目标命名、目录布局与实际写盘，core 仅保留“列可导出目标 + 渲染报告正文”的能力面。
* 收敛 `report export --all` 执行模型，统一改为“先列 canonical target，再逐个 render/write”，降低批量导出对旧写盘链路的耦合。
* 补齐 Windows CLI 关于 `exchange import` 配置覆盖语义与构建顺序的文档约束，明确 active converter config 覆盖后的运行态刷新由 core 负责完成。

### 修复 (Fixed)
* 修复月报导出文件名格式，统一从 `YYYYMM.ext` 调整为 `YYYY-MM.ext`，并保持 `202603` 与 `2026-03` 输入都导出为同一 canonical 文件名。
* 修复 `exchange export --help` / usage 对 `-o`、`--output` 输出路径要求提示不清的问题，改为展示完整导出调用形态。
* 修复 Windows CLI exchange 回归缺少“导入 `.tracer` 后立即按新 converter config 执行 `validate` / `query`”覆盖的问题，补齐 active config refresh 回归场景。
