## [v0.1.2] - 2026-04-02

### 新增功能 (Added)
* 新增 Windows Rust CLI `report render` 与 `report export` 的日期参数规则说明，明确 `day` 支持 `YYYYMMDD`/`YYYY-MM-DD`、`month` 支持 `YYYYMM`/`YYYY-MM`、`range` 支持 `start|end`，并在帮助文本中说明会先规范化为 ISO 再查询。
* 新增 `report render` 对空窗口查询结果的补充提示，当 `recent` 或 `range` 无记录时，额外输出 `start_date`、`end_date`、`requested_days`、`matched_day_count`、`matched_record_count`。
* 新增 Windows Rust CLI 对 runtime `report_window_metadata` 的承接与透传，确保 report 文本结果可以携带窗口统计信息。
* 新增 `reporting.target.not_found` 到 CLI 专用退出码 `ReportTargetNotFound` 的映射，便于脚本或上层调用区分“目标不存在”和普通逻辑失败。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/cli/windows/rust/src/commands/handlers/report/support.rs` 的 report 参数规范化链路，统一拆分 `render` 与 `export` 参数校验，集中处理 `day`、`month`、`week`、`year`、`recent`、`range` 的 canonical argument 生成。
* 重构 `apps/cli/windows/rust/src/commands/handlers/report/mod.rs` 与 runtime report client 返回类型，改为使用 `RenderedReport`/`ReportTextOutput` 承载正文和窗口元数据，而不是只返回纯文本字符串。
* 重构 `report export` 构建计划逻辑，改为在写盘前先生成已规范化的 render request，并补齐 `range` 导出类型与文件名生成规则。

### 修复 (Fixed)
* 修复 `report render day/month/range` 对紧凑日期输入直接透传的问题，改为在 CLI 层先规范化成 `YYYY-MM-DD`、`YYYY-MM`、`YYYY-MM-DD|YYYY-MM-DD` 后再发给 runtime。
* 修复 `report export day/month` 对紧凑日期输入查询参数和输出命名不一致的问题，改为统一使用规范化后的 target 参与查询与文件命名。
* 修复 `report render recent` 单个天数请求仍走 `days_list` 批量形态的问题，改为单值时发送 runtime 期望的 `type=recent` + `argument=<days>` 请求。
* 修复 `report export range` 缺少显式支持的问题，新增单次区间导出，并禁止 `report export range --all` 这种无法枚举目标的调用方式。
* 修复 CLI 未区分 report 目标不存在错误的问题，当前 `day/week/month/year` 缺失目标会返回专用退出码与包含 `reporting.target.not_found` 的错误详情。
* 修复日期参数校验过于宽松的问题，新增非法自然日、倒序 `range`、非正整数 `recent` 等输入的显式报错。
