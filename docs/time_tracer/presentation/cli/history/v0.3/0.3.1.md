## [v0.3.1] - 2026-04-15

### 技术改进/重构 (Changed/Refactor)
* 重构 Windows CLI `chart --type pie` 的查询语义，让其改为通过 `report-composition` 获取当前时间窗口内按 root 聚合的 breakdown 数据，不再复用 `report-chart` 的按天趋势数据。
* 重构 CLI pie 图的 HTML 渲染与模板，改为从 `slices[].root` 与 `slices[].duration_seconds` 生成扇区，并把统计摘要调整为适合 breakdown 图的 `Slices`、`Total (h)`、`Active roots` 与 `Range days`。
* 更新 `apps/cli/windows/AGENTS.md` 与 `apps/cli/windows/README.md` 的 chart 说明，明确 `line/bar/heatmap-*` 仍属于 trend 图，而 `pie` 现在表示选定 period 的 root breakdown。

### 修复 (Fixed)
* 修复 Windows CLI `chart --type pie` 仍允许 `--root` 的问题，现统一在参数校验阶段直接拒绝该组合，并给出 breakdown pie 使用整段时间窗口 root 集合的错误说明。
* 修复 `tools/suites/tracer_windows_rust_cli/tests/commands_query_data.toml` 对 `pie + --root` 黑盒用例的退出码断言，使其与当前 CLI `Invalid Arguments` 标准退出码保持一致。
* 保持 `tools/suites/tracer_windows_rust_cli` 黑盒 suite 与 `python tools/run.py verify --app tracer_windows_rust_cli --concise` 全量验证通过，确保 CLI pie 新语义与 suite golden 同步收敛。
