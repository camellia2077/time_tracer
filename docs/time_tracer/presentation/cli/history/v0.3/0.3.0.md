## [v0.3.0] - 2026-04-13

### 新增功能 (Added)
* 新增 CLI recent anchored export / render contract 支持，`report render recent 7 --as-of YYYY-MM-DD` 与 `report export recent 7 --as-of YYYY-MM-DD` 现在会直接携带 canonical `anchor_date`。

### 技术改进/重构 (Changed/Refactor)
* 重构 Windows Rust CLI reporting runtime 绑定，只加载 `tracer_core_runtime_temporal_report_json` 与 `tracer_core_runtime_report_batch_json`，并通过 temporal request JSON 统一处理 render、targets 与 export。
* 重构 `apps/cli/windows/rust/src/commands/handlers/report/support.rs`，用 `display_mode`、`selection_kind`、`date/start_date/end_date/days/anchor_date` 构造 canonical temporal payload，不再拼接 legacy `type/argument` 请求。
* 重构 `report export` 执行链路，让 single / all_matching / batch_recent_list 都直接走 temporal `operation_kind=export`，并把 targets 查询也改为通过 temporal reporting 入口完成。

### 修复 (Fixed)
* 修复 CLI 为了实现 recent 固定窗口而把 `--as-of` 本地改写成 `range` 的问题，避免宿主侧自行推导窗口后再伪装成另一种报告类型。
* 修复 reporting runtime loader 继续依赖 legacy `runtime_report_targets_json` 的问题，减少 reporting ABI surface 并让加载行为与 core canonical contract 一致。
* 保持 recent、range、month 等导出文件命名规则不变，同时修复 anchored recent `--all` 需要拆成多个 single export 的执行路径。

### 弃用/删除 (Deprecated/Removed)
* 删除 CLI 对 legacy reporting symbols 与旧 request 形状的依赖，包括 `runtime_report_targets_json` 与基于 `type/argument` 的 targets / export builder。
