# 基于每日数据的概率分析方向（按价值优先级）
## 5. 项目结构贡献度
- 目标：分析哪些项目路径对总时长贡献最大，并判断“结构是否合理”。
- 适配数据：project_tree、project_stats、total_duration。
- 价值：优化时间分配与项目层级。
- 示例：
  - Top-N 项目时长占比曲线，是否出现单点过高。

  ## 2. 睡眠与效率相关性
- 目标：分析“睡眠质量/时长”与“学习/产出”之间的统计关系。
- 适配数据：sleep_total_time、sleep_night_time、study_time、total_duration。
- 价值：直接回答“睡多/睡少是否更高效”。
- 示例：
  - 睡眠区间（<6h、6–7h、7–8h、>8h）对应的 study_time 均值/中位数。

## 1. 习惯稳定性与风险预警
- 目标：找出“最容易失控的习惯”和“连续风险天数”。
- 适配数据：status、exercise、sleep_total_time、getup_time、overnight。
- 价值：最直接提升自我管理的预警能力。
- 示例：
  - 连续 `status=0` 的概率分布、最长连续天数。
  - `exercise=0` 连续出现的概率、触发阈值（如连续 3 天）。



## 3. 运动与当日状态概率
- 目标：运动是否显著提高 `status=1` 的概率。
- 适配数据：exercise、status、total_exercise_time。
- 价值：帮助判断运动是否值得固定化。

## 4. 通宵行为模式
- 目标：通宵发生的前因后果。
- 适配数据：overnight（getup_time 为空/00:00）、sleep_night_time、次日 status。
- 价值：找到通宵的触发因素与成本。



## 6. 备注/活动文本信号
- 目标：从备注中识别高频情绪/事件，并与状态关联。
- 适配数据：day_remark、activity_remark。
- 价值：把主观日志转成可量化信号。
- 示例：
  - “压力/疲惫/加班”等关键词出现时，status=0 的概率变化。

## 7. 周期性与季节性
- 目标：识别星期/月份层面的波动。
- 适配数据：date、weekday、month。
- 价值：用于排程优化与资源配置。

---

## 最小可行分析集合（建议优先落地）
1. 连续失败天数分布（status/exercise）
2. 睡眠时长分组与学习时间均值
3. 通宵出现后次日状态概率
